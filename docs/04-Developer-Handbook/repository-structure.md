# Repository Structure

## Top-Level Layout

```text
facilityops-platform/
|- backend/
|- frontend/
|- docs/
|- infrastructure/
`- shared/
```

## Purpose of Each Area

- `backend/`: Django, DRF, tests, services, management commands
- `frontend/`: Next.js app, hooks, API client, types, shared UI
- `docs/`: architecture, task records, APIs, developer handbook, trackers
- `infrastructure/`: reserved infrastructure workspace
- `shared/`: reserved cross-application workspace

## Documentation Structure

- `docs/01-Architecture/`: stable architecture references
- `docs/02-Development/`: FO task delivery records
- `docs/03-API/`: API references
- `docs/04-Developer-Handbook/`: engineering process and governance
- `docs/development/`: live repository trackers and status pages
