# Project Status Guide

## Purpose

Define how to maintain `docs/development/project-status.md`.

## Update Triggers

Update project status when:

- the current task changes
- the current branch changes
- a module moves between status states
- a new FO task is completed
- a known issue materially changes
- a repository version or release marker changes

## Required Fields

- Project
- Current Phase
- Current Module
- Current Branch
- Completed Modules
- Completed FO Tasks
- Current Task
- Next Task
- Known Issues
- Last Reviewed Commit
- Last Merge
- Repository Version

## Source Priority

1. Current branch state
2. `docs/development/work-tree.md`
3. `docs/development/progress-map.md`
4. latest approved task context

## Maintenance Rule

Project status is a living summary, not an implementation log. Keep it current
and compact.
